=Instructions=
To open NeoTree explorer to the directory of the current file:
:Neotree reveal

To install plugin using Lazy.vim:
in ~/.config/nvim/init.lua:
  require("config.lazy")

~/.config/nvim/lua/config/lazy.lua
Create plugin specs in ~/.config/nvim/lua/plugins
Each file should return a table with the plugins you want to install

LazyVim extras are specified in ~/.config/nvim/lazyvim.json, but these can be toggled by running nvim with no arguments and selecting "x" from the menu.

CMake tools can be enabled from Lazy Extras.
To disable formatting of CMake files in a directory, create a file in that directory named .clang-format with this contents:
BasedOnStyle: Google
DisableFormat: true
or add comment to code // clang-format: off
See neocmake and cmake formatters

To check on formatter:
:Conform
Log is in ~/.local/state/nvim/conform.log

Language support is done with Mason "<space>cm"

Python editing intermittently freezes.  To disable Python support, use :set filetype=txt

To diagnose problems with LSP (file or type not found):
~/.local/share/nvim/mason/bin/clangd --check=MK2/TOOLS/gddgen/messages/s0e09_ac_enable_drop_system_track_mt_gdd/main.cpp
This will show where compile_commands.json is loaded from and any errors trying to parse the file given by --check.
CLang is searching for compile_commands.json in the build directory and ignoring build/..
If CMake doesn't generate build/compile_commands.json, set the environment variable CMAKE_EXPORT_COMPILE_COMMANDS=enable
